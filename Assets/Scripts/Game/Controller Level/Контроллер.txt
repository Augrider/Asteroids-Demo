Цель - быть промежуточным звеном между правилами, интерфейсом и плагинами
Контроллер хранит в себе ссылки на соответвующие плагины, собирает цикл игры, обрабатывает данные управления и посылает данные интерфейсу
Для изоляции контроллера от изменений в правилах нужно определить интерфейс/абстракцию для правил в самих правилах, а на самом высоком уровне добавить правила в контроллер (или добавить с помощью Initializer). Сам контроллер не будет знать о правилах (internal) и будет работать с ними через интерфейс (Context интерфейсы нужны для работы ввода-вывода правил)
